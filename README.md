# Clara AI Automation Pipeline

This project implements a **zero-cost automation pipeline** that converts conversation transcripts from **Demo** and **Onboarding** calls into structured **Retell AI Voice Agent configurations**.

The system extracts operational rules from conversations and generates **version-controlled AI agent specifications**.

---

# Overview

Service businesses such as electrical contractors, HVAC providers, and fire protection companies receive many types of customer calls:

* emergency service calls
* non-emergency service requests
* inspection scheduling
* after-hours calls

Each business has different rules for:

* business hours
* emergency definitions
* call routing
* transfer protocols

This pipeline converts:

**Human Conversations → Structured AI Agent Configurations**

# Input Data

The pipeline uses **Fireflies transcript JSON files**.

Example:

```text
dataset/inputs/demo/bens_electric_demo.json
dataset/inputs/onboarding/bens_electric_onboarding.json
```

These files contain:

* call transcript
* speaker labels
* timestamps
* conversation content

Using transcripts avoids the need for **paid speech-to-text services**, keeping the pipeline **zero-cost**.

# Pipeline A — Demo Call → Agent v1

Input:

```text
dataset/inputs/demo/<demo_call>.json
```

Steps:

1. Parse transcript
2. Extract business information
3. Generate account memo
4. Generate Retell agent specification
5. Save results as **version v1**

Output location:

```text
outputs/accounts/<account_id>/v1/
```

Generated files:

```text
account_memo.json
agent_spec.json
```

---

# Pipeline B — Onboarding Call → Agent v2

Input:

```text
dataset/inputs/onboarding/<onboarding_call>.json
```

Steps:

1. Load existing **v1 memo**
2. Extract updated rules
3. Merge updates with previous configuration
4. Generate updated agent specification
5. Generate `changes.md` file

Output location:

```text
outputs/accounts/<account_id>/v2/
```

Generated files:

```text
account_memo.json
agent_spec.json
changes.md
```

# Running the Project

### 1 Install dependencies

```bash
pip install -r requirements.txt
```

---

### 2 Set up Environment Variables (Optional but Recommended)

By default, the pipeline uses a rule-based fallback for extraction. To execute true dynamic extraction on any untested transcript cases, please provide a free Groq API key:

1. Copy `.env.example` to `.env`
2. Add your key: `GROQ_API_KEY="your_key_here"`

---

### 3 Add transcript files

Place transcripts inside:

```text
dataset/inputs/demo/
dataset/inputs/onboarding/
```

Example:

```text
dataset/inputs/demo/bens_electric_demo.json
dataset/inputs/onboarding/bens_electric_onboarding.json
```

---

### 4 Run the pipeline

```bash
python pipeline.py
```

---

outputs/accounts/bens_electric/v1/
outputs/accounts/bens_electric/v2/

# Design Goals

This system was designed to be:

* **Zero-cost**
* **Modular**
* **Reproducible**
* **Version-controlled**
* **Batch capable**

---

# Limitations

* Supports **Groq LLM API** (e.g. Llama 3.1) for dynamic extraction
* Gracefully falls back to pure Python rule-based logic if no `.env` API key is provided
* Transcript formats may vary depending on Fireflies export structure
